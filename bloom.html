Bloom

Building a mobile app that helps gardeners manage their gardens

Type Individual Project
Role Lead Developer
Tech React Native, Expo, Firebase (NoSQL), Git, Figma
Timespan Fall 2020 | 4 weeks

Summary

Problem

Gardeners, especially beginners, can be easily overwhelmed by the responsibilities involved with starting and maintaining a garden. Specifically, knowing which plants are suited to growing in their area, and then monitoring the progress of those plants (after they've been planted), can be considerable obstacles for gardeners.

Actions

I designed and developed a simple mobile app that enables users to discover which plants are suitable to their location (based on zipcode), add them to their virtual garden, then monitor the plants' progress by adding notes and photos.

Results

An MVP of the app has been built, and is currently undergoing iterative impovements before being deployed.

Bloom GitHub repository

Background

Many members of my family are avid gardeners. Since they are fairly geographically dispersed, they are limited in the relevant gardening knowledge they can share with eachother. Plants that grow well in one region may not grow well (or at all) in another. Additionally, many of them take notes about their gardens on paper and experience difficulty organizing and referencing these notes.

The United States Department of Agriculture (USDA) assigns hardiness zones to regions of the United States so that farmers (and gardeners) can better understand how well different plants will grow in a specific region. It's relatively easy to determine which hardiness zone you live in; however, it's difficult to determine which plants are suitable to your hardiness zone. There is no centralized database of all the plants and their hardiness zones.

For the final project of my SI 669 - Developing Mobile Experiences course, I designed and built a mobile app that provides a list of plants that are suitable to the user's hardiness zone. Additionally, users can create a virtual representation of their garden(s) and add their plants to it. Finally, they can add notes and images to their gardens' plants to help monitor their progress and even help plan for the next year.

Backend

For the purposes of this app, I essentially needed to create a series of nested one-to-many relationships (users -> gardens -> plants -> notes). This led to me explore NoSQL databases as possible backend solutions. Ultimately, I found Google Firebase and decided that its Firestore Database service fit the needs of this project perfectly.

Firestore is a NoSQL database composed of collections and documents. The collections are essentially arrays, while the documents act as objects. For this project, I used three primary collections: Users, Plants, and Zipcodes. However, the Users collection contains a number of nested collections and documents. The Plants and Zipcodes collections only contain documents.



Bloom Database Diagram


Firebase provides a number of backend features that facilitate basic CRUD functionality, as well as more advanced features such as subscriptions. Although this setup made development simple and easy, it also created some headache when trying to implement custom backend functionality (e.g. duplicate username prevention). Much of this functionality had to be moved to the frontend.

Frontend

The app's frontend was built in React Native and used the Expo framework to tap into native features, such as the camera. In order to communicate with the backend, it uses the Firebase SDK, which provides a number of built-in CRUD functions for both the Firestore database and the Firebase Storage service (which is used to store user photos).

To start, I created a rough series of low-fidelity wireframes to demonstrate the app's user flow.

Bloom Wireframes

I then created a basic React Native app and built out the necessary functionality. For the data model, I used the simpleton pattern to reduce complexity. Next, I styled the the screens and their elements. The functionality of the app was relatively simple to develop; however, this was my first experience using React Native's styling functionality. Using flex properly within React Native can be frustrating and took awhile to fully understand.

Next Steps

Some of the top priorities moving forward include:

  • Improve the styling on the app's frontend
  • Improve the UX of the login process
  • Add more plants to the database
  • Add stock images for all of the plants
  • Provide an option to automatically determine the user's hardiness zone based on their phone's GPS location
  • Enable the user to set up reminders through the app (e.g. water the tomatoes)
  • Provide more advanced filtering/sorting options for plants
