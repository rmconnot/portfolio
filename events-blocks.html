Events Blocks

Developing customizable blocks that automatically pull up-to-date information about Michigan Engineering events

Type Team/Individual Project
Role Lead Developer
Tech React.js, PHP, Docker, Git, REST, WordPress, Figma
Timespan Fall 2020 | 12 weeks

Summary

Problem

Michigan Engineering needed a way to inform students about upcoming College events on their websites.

Actions

I worked with a cross-functional team to research, design, and develop two modular, customizable blocks. One maintains an up-to-date feed of upcoming events (which can be filtered based on various criteria), and a second displays a card for an individual event.

Results

While the development of the overall design system is still ongoing, the events blocks are complete and are currently being used on Michigan Engineering websites.

Background

With the introdcution of the Gutenberg Editor for WordPress, Michigan Engineering made the decision to create a new design system for the College of Engineering. This design system, and the functional components it contains, affects all of the College's websites (>100 individual sites). Nearly all of these websites need a way to pull information about upcoming College events and display them to users.

The purpose of Gutenberg is to provide users with an intuitive, modular, WYSIWYG-like experience for building and editing pages. This necessitates the use of modular components (blocks) that users can add to a page, edit, and display on the frontend. Gutenberg blocks are primarily built using React.js; however, dynamic blocks (where the block changes in reaction to certain conditions) also need to be developed in PHP.

Events Feed Block

The Events Feed Block is a dynamic block that takes user input and creates a visual feed of College events pulled from Happening@Michigan. The displayed events are filtered based on the provided input.

Events data is cached by utilizing WordPress Transients. This prevents a fresh API call from being made every time a user opens a page with an Events Feed Block present. Data received from a previous API call is stored for 1 hour, after which a new API call is made to retrieve up-to-date events data.

A new Events Feed Block in the editing interface
Editing form


Users are able to specify a number of options when creating a new events feed block. These options primarily filter the events, but they can also change how the events are displayed.

All college units have a sponsor ID which can be used to filter the upcoming events that are displayed. Users can also provide additional filtering criteria such as the type of events and the tags associated with them.

The block's appearance can also be changed to meets the needs of the users. These options were the result of user research conducted several months in advance.

Single Event Block

The Single Event Block takes user input in the editing interface, retrieves data about a specified event from Happening@Michigan, and displays an event "card" on the frontend.

Users can specify the event they would like to retrieve, as well as how it is displayed. The color picker provides color options that are consistent with the University of Michigan's color palette.
